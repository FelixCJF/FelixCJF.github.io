设计模式之代理模式

Java设计模式
  • 代理模式

定义：代理模式是对象的结构模式。代理模式给某一个对象提供一个代理对象，并由代理对象控制对原对象的引用。

类图


抽象对象：声明了代理对象和被代理对象的共同接口，这样一来可以随时用代理对象来代理被代理对象
被代理对象：定义了被代理对象
代理对象:代理对象内部定义了被代理对象的引用，方便操作被代理对象。代理对象提供一个与被代理对象相同的接口，以便可以在任何时候替代目标对象。代理对象通常在客户端调用传递给目标对象之前或之后，执行某个操作，而不是单纯地将调用传递给目标对象。
AOP思想就是基于代理模式

代码实现代理卖票

  1. 先定义一个卖票系统接口，所有地方卖票都得实现这个系统 

    package com.felix.proxy;
/**
 * 代理对象和目标对象的共同接口
 * 卖票系统
 *
 */
interface AbstractObject {
public void queryTicket();// 查询票
public void buyTicket();//买票
}

  2. 定义火车站卖票，里面有卖票和查询功能 

    package com.felix.proxy;
/**
 * 真正的目标对象
 * 火车站买票
 *
 */
public class ReallyObject implements AbstractObject{

public void queryTicket() {
System.out.println("在火车站查询票");
}
public void buyTicket() {
System.out.println("在火车站买票");
}
}

  3. 定义代售点 

    package com.felix.proxy;
/**
 * 代理类
 * 火车票代售点
 *
 */
public class ProxyObject implements AbstractObject {

private AbstractObject abstractObject;

//默认作为火车站的代理
public ProxyObject(){
abstractObject = new ReallyObject();
}
public ProxyObject(AbstractObject abstractObject){
this.abstractObject = abstractObject;
}

public void queryTicket() {
//表面上是代售点查的票，实际上还是在火车站查的
this.abstractObject.queryTicket();
}

public void buyTicket() {
//表面上是代售点买的票，实际上还是在火车站买的
abstractObject.buyTicket();
}
}

  4. 测试类

    package com.felix.proxy;

public class ProxyTest {
public static void main(String[] args) {
//创建代理对象
ProxyObject proxyObject = new ProxyObject();
//执行买票
proxyObject.queryTicket();
proxyObject.buyTicket();
}
}

总结

代理模式使用了Java的多态，干活的是被代理类也就是目标类，代理类主要负责接后，然后把需要干的事交给幕后。只需要要求代理类和被代理类实现共同的接口。
